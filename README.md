# Домашнее задание №3. Docker

Репозиторий содержит решение для автоматизации генерации и анализа данных с использованием Docker-контейнеров и bash-скриптов.

**Тема проекта:** Генерация и анализ данных базы фотографий.

---

## Структура проекта

* `generator/` — контейнер на базе Python для создания сырых данных.
* `reporter/` — контейнер на базе Node.js для анализа данных и генерации HTML-отчета.
* `data/` — общая директория на хосте для обмена данными между контейнерами через монтирование томов.
* `run.sh` — главный скрипт управления контейнерами.

---

---

## Работа с командами
1. Работа с Генератором данных
* `./run.sh create_local_data` - cоздает данные напрямую на хосте без использования Docker в изолированной папке
* `./run.sh build_generator` - собирает Docker-образ photo_generator на основе python:3.11-slim
* `./run.sh run_generator` - запускает контейнер, который генерирует файл data.csv и сохраняет его в локальную папку data/ на хосте с помощью монтирования томов

2. Работа с Аналитиком данных
* `./run.sh build_reporter` - собирает Docker-образ photo_reporter на основе node:20-alpine, автоматически устанавливая зависимости через npm install на этапе сборки
* `./run.sh run_reporter` - запускает контейнер, который считывает сгенерированный data.csv из примонтированной папки, анализирует его и сохраняет итоговый отчет report.html обратно в папку data/

3. Вспомогательные команды
* `./run.sh structure` - отображает дерево каталогов и файлов текущего состояния репозитория
* `./run.sh clear_data` - безопасно удаляет результаты генерации и анализа (.csv и .html файлы) из папки data/, оставляя саму директорию готовой к повторному циклу
* `./run.sh inside_generator ./run.sh inside_reporter` - позволяют убедиться, что контейнеры корректно видят примонтированную папку data/ с хоста
